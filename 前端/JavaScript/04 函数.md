我们经常需要在脚本的许多地方执行很相似的操作。

例如，当访问者登录、注销或者在其他地方时，我们需要显示一条好看的信息。

函数是程序的主要“构建模块”。函数使该段代码可以被调用很多次，而不需要写重复的代码。

内建函数的示例如 `alert(message)`、`prompt(message, default)` 和 `confirm(question)`。我们也可以创建自己的函数。

## [函数声明](https://zh.javascript.info/function-basics#han-shu-sheng-ming)

使用 **函数声明** 创建函数。

看起来就像这样：
```javascript
function showMessage() {
  alert( 'Hello everyone!' );
}

showMessage();
showMessage();
```

调用 `showMessage()` 执行函数的代码。这里我们会看到显示两次消息。

## [局部变量](https://zh.javascript.info/function-basics#ju-bu-bian-liang)

在函数中声明的变量只在该函数内部可见。

例如：
```javascript
function showMessage() {
  let message = "Hello, I'm JavaScript!"; // 局部变量

  alert( message );
}

showMessage(); // Hello, I'm JavaScript!

alert( message ); // <-- 错误！变量是函数的局部变量
```

## [外部变量](https://zh.javascript.info/function-basics#wai-bu-bian-liang)

函数也可以访问外部变量，如果在函数内部声明了同名变量，那么函数会 **遮蔽** 外部变量。例如，在下面的代码中，函数使用局部的 `userName`，而外部变量被忽略：

```javascript
let userName = 'John';

function showMessage() {
  let userName = "Bob"; // 声明一个局部变量

  let message = 'Hello, ' + userName; // Bob
  alert(message);
}

// 函数会创建并使用它自己的 userName
showMessage();

alert( userName ); // John，未被更改，函数没有访问外部变量。
```

	全局变量
	
	任何函数之外声明的变量，例如上述代码中的外部变量 `userName`，都被称为 全局 变量。
	
	全局变量在任意函数中都是可见的（除非被局部变量遮蔽）。
	
	减少全局变量的使用是一种很好的做法。现代的代码有很少甚至没有全局变量。大多数变量存在于它们的函数中。但是有时候，全局变量能够用于存储项目级别的数据。


## [参数](https://zh.javascript.info/function-basics#can-shu)

## 1. 什么是参数？

参数就是你**传给函数的数据**，让函数知道要处理什么东西。

想象一下，函数是一个**奶茶店**：

```
点单（口味, 甜度）   ← 这就是参数
```

````javascript
function 点单(口味, 甜度) {
  console.log("你点了：" + 口味 + "，" + 甜度)
}

点单("珍珠奶茶", "三分糖")   // 你点了：珍珠奶茶，三分糖
点单("柠檬茶", "全糖")       // 你点了：柠檬茶，全糖
````

- `口味` 和 `甜度` 就是**参数**
- `"珍珠奶茶"` 和 `"三分糖"` 就是你传进去的**具体值**

## 2. 少传参数会怎样？

如果函数需要 2 个参数，你只传了 1 个，没传的那个就是 `undefined`（未定义）。

````javascript
function 打招呼(名字, 消息) {
  console.log(名字 + "说：" + 消息)
}

打招呼("小明", "你好")    // 小明说：你好      ✅ 正常
打招呼("小明")           // 小明说：undefined  😅 消息没传，变成 undefined
````

---

## 3. 默认值：给参数一个"备胎"

为了避免 `undefined`，你可以设置一个**默认值**——如果没传，就用这个：

````javascript
function 打招呼(名字, 消息 = "你好呀") {
  console.log(名字 + "说：" + 消息)
}

打招呼("小明", "早上好")  // 小明说：早上好     ← 传了，用传的
打招呼("小明")           // 小明说：你好呀     ← 没传，用默认值
````

就像奶茶店：

> "甜度不说的话，默认给你做半糖。"

---

## 4. 旧代码中的默认值写法

教程提到了两种**老写法**，现在不推荐但你可能会遇到：

### 写法一：手动检查 undefined

````javascript
function 打招呼(名字, 消息) {
  // 如果消息没传（是 undefined），就给个默认值
  if (消息 === undefined) {
    消息 = "你好呀"
  }
  console.log(名字 + "说：" + 消息)
}
````

逻辑很简单：**没传？那我自己赋一个。**

### 写法二：用 `||`（或运算符）

````javascript
function 打招呼(名字, 消息) {
  消息 = 消息 || "你好呀"
  console.log(名字 + "说：" + 消息)
}
````

**`||` 的规则是：左边是"假值"就用右边的。**

什么是"假值"？这些都是：

| 假值 | 说明 |
|------|------|
| `undefined` | 没传参数 |
| `null` | 空值 |
| `""` | 空字符串 |
| `0` | 数字零 |
| `false` | 布尔假 |

所以 `消息 || "你好呀"` 的意思是：

```
如果 消息 是假值（包括 undefined、""、0 等）→ 用 "你好呀"
如果 消息 有正常值                            → 用 消息 本身
```

### ⚠️ 这个老写法有坑！

````javascript
消息 = 消息 || "你好呀"

// 如果你故意传了空字符串 ""
打招呼("小明", "")  // 小明说：你好呀  ← 你传了 "" 但被当成假值，被覆盖了！
````

你本来想让消息为空，但 `||` 觉得空字符串是"假值"，帮你换成了默认值。

### ✅ 现代写法：直接用默认参数语法

````javascript
function 打招呼(名字, 消息 = "你好呀") {
  console.log(名字 + "说：" + 消息)
}

打招呼("小明", "")  // 小明说：        ← 正确！空字符串不会被覆盖
打招呼("小明")      // 小明说：你好呀   ← 只有真的没传才用默认值
````

## 📝 总结

| 概念 | 一句话解释 |
|------|-----------|
| 参数 | 传给函数的数据，像点单时告诉店员要什么 |
| 复制品 | 函数拿到的是复印件，改了不影响原件 |
| 少传参数 | 没传的变成 `undefined` |
| 默认值 `= xxx` | 没传就用这个值，推荐用这种写法 |
| `\|\|` 老写法 | 能用但有坑，空字符串和 0 会被误杀 |
