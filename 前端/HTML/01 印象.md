HTML 存在于扩展名为 `.html` 的文本文件中

最常见的 HTML 文件是 `index.html`，它通常用于包含网站主页的内容。同样常见的是看到带有各自 `index.html` 的子文件夹，因此一个网站可以在不同的位置有多个索引文件。

元素：
开始标签、内容和结束标签
「词典」：[HTML 元素参考 - HTML（超文本标记语言） | MDN](https://developer.mozilla.org/zh-CN/docs/Web/HTML/Reference/Elements)

嵌套：类似括号的结构

空元素：
并非所有元素都遵循开始标签、内容和结束标签的模式。有些元素只包含一个标签，通常用于在文档中插入/嵌入某些东西。这类元素被称为空元素。例如：
```
<img
  src="https://raw.githubusercontent.com/mdn/beginner-html-site/gh-pages/images/firefox-icon.png"
  alt="Firefox 图标" />
```

元素的属性：
例如上文的src=...，alt=...

一个属性应该有：
- 一个在它和元素名之间的空格。（对于拥有多个属性的元素，属性之间也应该用空格分隔。）
- 属性名，后跟一个等号。
- 一个属性值，用开始和结束的引号包裹。

布尔属性：
不写就false，写了（不管值是什么）就true
```
<!-- 使用 disabled 属性来防止终端用户输入文本到输入框中 -->
<input type="text" disabled="anything" />

<input type="text" disabled />

<!-- 下面这个输入框不包含 disabled 属性，所以用户可以向其中输入 -->
<input type="text" />
```

省略属性值周围的引号：
某些情况下允许，最好还是加上。

引号内使用引号：
如双引号用&quot;
「词典」：[字符参考 - MDN Web 文档术语表：Web 相关术语的定义 | MDN](https://developer.mozilla.org/zh-CN/docs/Glossary/Character_reference)

剖析完整文档：
```
<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8" />
    <title>我的测试页面</title>
  </head>
  <body>
    <p>这是我的页面</p>
  </body>
</html>
```

<!doctype html>。历史遗留，就这么写。

<html> 根元素，置于最前最后。

<head> 这个元素充当一个容器，用于存放你想包含在 HTML 页面中但不向页面浏览者显示的内容。这包括会出现在搜索结果中的关键字和页面描述、用于样式化内容的 CSS、字符集声明等等。
	“容器”是用于封装、分组和组织其他HTML元素（如文字、图片、列表）的特殊元素



<meta>杂物口袋。其中如charset 属性可将文档的字符编码指定为 UTF-8，它包含了绝大多数人类书面语言的字符。通过这个设置，页面现在可以处理它可能包含的任何文本内容。没有理由不设置它，而且它可以帮助避免以后的一些问题。

<title>设置页面标题，这个标题会出现在加载该页面的浏览器标签页中。当页面被收藏为书签时，页面标题也用于描述该页面。

<body>它包含了在页面上显示的所有内容，包括文本、图像、视频、游戏、可播放的音轨或其他任何东西。

空白：
无论长度为多少的空白序列均解释为一个空格

注释：

```
<!-- <p>注释</p> -->
```
